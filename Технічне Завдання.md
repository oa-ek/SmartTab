Технічне завдання (ТЗ) на розробку інтернет-магазину комп'ютерної техніки "SmartTab"

1. Загальний опис проєкту
Мета проєкту: Розробка повноцінного вебдодатку для продажу комп'ютерних комплектуючих та готових ПК (з можливістю їх кастомізації).
Система повинна забезпечувати зручний клієнтський інтерфейс (каталог, конфігуратор, кошик) та сучасну, швидку панель управління для адміністраторів (автоматизований складський облік серійних номерів, управління виробниками та категоріями). Дизайн адмін-панелі має бути наближеним до сучасних маркетплейсів.

2. Архітектура та технологічний стек
Проєкт розробляється на базі N-рівневої архітектури (N-Tier) із розділенням на шари: Core (Сутності), Data (База даних), Services (Бізнес-логіка) та UI (Веб-інтерфейс + REST API для адмінки).

Бекенд: C#, ASP.NET Core MVC, Web API (для асинхронних запитів інтерфейсу).

База даних: MS SQL Server, Entity Framework Core.

Фронтенд: HTML, Razor Views, Tailwind CSS (для швидкої та гнучкої стилізації), Vanilla JavaScript (Fetch API для динамічного оновлення UI без перезавантаження сторінки).

3. Ролі користувачів

Гість (Анонімний користувач): Може переглядати каталог, користуватися фільтрами, додавати товари в кошик.

Авторизований клієнт: Має доступ до особистого кабінету, історії замовлень (з фіксацією цін на момент покупки) та збереженого кошика.

Адміністратор: Має повний доступ до CRUD-операцій каталогу (товари, категорії, виробники), управління замовленнями та складським обліком.

4. Функціональні вимоги
4.1. Панель адміністратора (Керування каталогом)

SPA-подібний інтерфейс (Single Page Application) для керування сутностями без перезавантаження сторінки.

Управління товарами (CRUD для Products), зручна прив'язка до категорій та виробників.

Підтримка динамічних характеристик (через зв'язану таблицю ProductSpecifications або JSON) залежно від обраної категорії товару.

Управління списком категорій (Categories) та виробників (Manufacturers).

Завантаження, попередній перегляд та збереження зображень товарів на сервері.

4.2. Складський облік (Inventory Management)

Автоматична генерація унікальних серійних номерів під час створення товару на основі заданої "Початкової кількості" (initialQuantity).

Можливість швидкого додання нових серійних номерів до вже існуючого товару.

Точний облік кожної фізичної одиниці товару в таблиці InventoryItems.

Прив'язка конкретного серійного номера до замовлення клієнта під час відвантаження товару (захист від гарантійного шахрайства, фіксація статусу IsSold).

4.3. Клієнтська частина (Каталог)

Вивід списку товарів із пагінацією та пошуком.

Детальна сторінка товару з фотографією, описом, відображенням фактичної наявності на складі (StockCount), ціною та повними характеристиками.

4.4. Конфігуратор ПК (Customization Feature)

Можливість обрати базовий готовий ПК (тип товару Built PC).

Інтерфейс для заміни стандартних деталей на альтернативні комплектуючі з каталогу.

Динамічний перерахунок фінальної ціни збірки.

Збереження обраних опцій конфігуратора в кошик та історію замовлень.

4.5. Кошик та Оформлення замовлення

Зберігання кошика в базі даних для синхронізації між пристроями авторизованого користувача.

Оформлення замовлення зі створенням фінального "чеку" (сутності Orders, OrderItems).

Жорстка фіксація ціни товару в момент покупки (UnitPrice), що гарантує незмінність історії замовлень при майбутніх змінах цін у каталозі.

5. Етапи розробки 

Етап 1 (Фундамент та Адмінка): Проєктування схеми БД (EF Core), застосування міграцій. Створення REST API контролерів. Розробка динамічного UI адмін-панелі (Tailwind + JS) для сутностей Products, Categories, Manufacturers. Реалізація завантаження фото та автоматичної генерації серійних номерів.

Етап 2 (Клієнтський каталог та Авторизація): Створення клієнтської сторінки магазину, виведення карток товарів. Налаштування системи реєстрації та логіну (Identity).

Етап 3 (Бізнес-логіка, Кошик, Конфігуратор, Замовлення та Склад): Реалізація шару SmartTab.Services. Розробка логіки додавання товарів до кошика та функціоналу заміни комплектуючих у готових збірках (конфігуратор). Реалізація процесу чекауту (перетворення кошика на замовлення), а також розробка інтерфейсу комірника для списання відвантажених товарів за їхніми унікальними серійними номерами (InventoryItems).
